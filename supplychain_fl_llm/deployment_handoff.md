# Multi-Laptop Deployment Handoff

## Keep and Run on Server Laptop

Keep and run these folders on the server laptop:
- central_server
- blockchain
- dashboard
- experiments

Keep shared utilities:
- utils

Keep generated runtime outputs:
- artifacts
- logs

Keep full synthetic datasets:
- data/generated

## What to Send to Each Client Laptop

Send these folders/files to each client laptop:
- node_client
- utils
- data (needed for local fallback generation imports)
- data/generated (only the partner-specific CSV for that client)
- artifacts (must include the deployed contract artifact generated by blockchain/deploy_contract.py)
- requirements.txt
- .env.example

### Partner Data Mapping Example

- Client 1 gets: data/generated/partner_1.csv
- Client 2 gets: data/generated/partner_2.csv
- Client 3 gets: data/generated/partner_3.csv

## What Clients Do Not Need

Clients do not need:
- central_server
- dashboard
- experiments
- blockchain deployment scripts (they only need the deployed contract artifact from the server)
